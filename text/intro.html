Introduction¶

This text is the companion guide to a curriculum for creating performances with robots. It is written to be accessible to middle-school students, but could be used at a variety of levels.

The essential skill is to understand the meaning of movement, action, and form in context well enough to convey a performance idea via a machine. Accomplishing this using a robot requires finding insight into performance by robot proxy as well as learning analytic and programming skills.

The text is written for the Finch Robot platform sold by BirdBrain Technologies, and uses the Snap! visual programming system. However, the core ideas are general and this course could be ported to other languages and robots.

Prerequisites¶

The text assumes that students are already familiar with the basic use of Snap!. Students already familiar with Scratch may need to study a few of the additional features provided by Snap.

The exercises assume students are already familiar with these basic programming ideas in Snap:

  1. imperative programming as sequences of commands
  2. data representation as numbers and strings (atoms)
  3. the idea of composite data representation as lists
  4. creating functions with arguments
  5. state machine transition diagrams and implementation

And these topics related to robotics:

  1. issuing robot commands to change physical state (wheel velocity, LED color)
  2. controlling timing of actions using delays
  3. sampling robot sensors to estimate physical state (proximity sensors, accelerometer)

Learning Goals¶

The following topics include the essential learning goals related to robot performance:

  1. capturing human performance via robot
  2. improvising with robots
  3. formulating an interaction (pas de deux or dialogue)
  4. the role of rhythm
  5. ensemble choreography
  6. creating meaning through context: story, setting, costuming
  7. writing a script for video-recorded performance
  8. writing a script for live performance

Accomplishing these objectives will require learning more fundamentals of robotics including the following:

  1. using data to represent actions and trajectories
  2. using a networked key-value store
  3. using a distributed protocol
  4. graphical analysis of the convex hull
  5. the definition of uncertainty
  6. graphical reasoning about capture regions
  7. creating primitives for sensorless navigation
  8. designing environmental features for navigational assistance

Implementing these ideas will require new programming skills including the following:

  1. using lists as structured data: arrays, association lists
  2. essential functional programming: values, iteration, and mapping
  3. understanding recursive functions

Daily Sessions¶

The course involves a series of hour-long sessions mixing technical exercises to build robot skills and drama exercises to develop storytelling abilities. The course will loosely alternate between technical and drama topics, the exact schedule is yet to be determined.

Summary of Drama Sequence¶

  1. Introductions, warm-ups, and improv exercises.
  2. Storytelling with objects: finding a story in anything, seeing from a new point of view.
  3. Storytelling with robots: movement, setting, and relationships.
  4. Robot dance party. Rhythm and synchrony. Ensemble choreography.
  5. Creating a character, finding a voice.
  6. Costuming and set design.
  7. Robot circus.
  8. Developing a theme.
  9. Rehearsal skills.
  10. Scriptwriting workshop.
  11. Practice, practice, practice.
  12. Documentation.

Summary of Technical Sequence¶

  1. Introduction to Snap!
  2. State machine programming. Shared autonomy.
  3. Functions, lists, and data.
  4. Movement primitives. Parameterization.
  5. Robot geometry. Diff-drive kinematics, center of rotation, convex hull.
  6. Sensorless navigation. Uncertainty, collisions and capture regions.
  7. Ensemble synchronization (networking).
  8. Distributed scripting and protocols.
